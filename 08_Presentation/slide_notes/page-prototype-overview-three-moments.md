# Three Prototype Moments

## Source visual

![[page-prototype-overview-three-moments.svg]]

## Slide purpose

Make the audience understand the three exhibition-ready prototype moments at a glance.

This slide positions the prototypes as connected moments in a service package, not three separate interface concepts.

## Key message

The prototype system moves from **Detect** to **Review** to **Learn**:

- Detect: make AI-human disagreement visible without blame.
- Review: support human-led arbitration and preserve handover context.
- Learn: turn difficult cases into non-blame learning memory and calibration material.

## Prototype moments

### Prototype 1 — Disagreement Detection Cue

Question it answers: Where does the disagreement become visible?

A soft, non-blame cue appears inside the existing reading workspace when human judgement and AI evidence layer do not align.

Key artefacts:
- Collapsed soft cue
- Expanded comparison review
- Disagreement type label

### Prototype 2 — Arbitration Support Layer

Question it answers: How is the case handed over and reviewed?

A warm handover and pause mechanism is embedded in the existing review workspace to support human-led review and arbitration rationale capture.

Key artefacts:
- Warm Handover Summary
- Arbitration Pause Point
- Rationale capture
- Learning handoff

### Prototype 3 — Decision Receipt & Learning Memory

Question it answers: How does one difficult case become future learning?

A non-blame decision receipt is generated after arbitration as service evidence for audit, review and learning, then the case becomes a reusable learning memory for calibration.

Key artefacts:
- Non-blame Decision Receipt
- Learning Memory Card
- Calibration case
- Review workshop material

## Speaker notes draft

This overview slide makes the service package legible.

The project is not replacing the existing clinical review environment. It is prototyping three disagreement support moments that sit around the existing reading and review environment. First, the disagreement becomes visible through a soft cue inside the existing reading workspace. Second, the case moves into a warm handover and human-led senior review through an arbitration support layer. Third, the final recall decision and arbitration rationale become a non-blame learning record that can support calibration and future review.

Together, these moments show how AI-human disagreement can be detected, resolved and learned from without treating AI as the recall decision-maker or treating human judgement as a source of individual blame.
