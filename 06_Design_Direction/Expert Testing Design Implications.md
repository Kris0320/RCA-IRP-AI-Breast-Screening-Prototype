# Expert Testing Design Implications

## 1. Shift in Framing

From:
AI-as-second-reader disagreement workflow

To:
A lightweight support layer for difficult AI-human cases across different AI-assisted breast screening workflows.

Sophia Zackrisson's feedback suggests that the project should not depend on one fixed AI placement model. AI may be used as a second reader, as triage / risk stratification, or as part of consensus review. The design value is therefore not in proposing one new workflow, but in supporting difficult AI-human comparison moments across different workflows.

## 2. Design Principles After Expert Testing

- Workflow-sensitive.
- Low-burden.
- Auto-capture first, confirm only when necessary.
- Prior-image-aware.
- Human-led consensus review.
- Outcome-linked learning.
- Embedded in existing workstations.

## 3. Prototype Changes

- Rename arbitration as consensus / difficult-case review where appropriate.
- Keep soft cue but make it non-alarm-like.
- Make comparison review lightweight.
- Replace heavy manual cards with suggested reason tags.
- Keep receipt but make it non-blame and outcome-linked.
- Move case library into scheduled audit / teaching / review sessions.
- Downgrade tea-room / wall / daily-space interventions.
