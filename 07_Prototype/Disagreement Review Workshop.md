# Disagreement Review Workshop

## One-line concept

A recurring, non-blame case review session where selected AI-human disagreement cases from arbitration are transformed into shared learning, calibration and future service improvement.

## Role in the service system

AI-human disagreement detected  
-> Arbitration / senior review  
-> Non-blame Decision Receipt  
-> Learning Memory Card  
-> Disagreement Review Workshop  
-> Updated practice / calibration / protocol refinement

Arbitration resolves the individual case. The workshop helps the team learn from how it was resolved.

The workshop sits after case-level resolution. It does not reopen the final recall decision. Instead, it asks what the disagreement pattern, evidence trail and arbitration rationale can teach the wider team.

## Why this is needed

Current arbitration may resolve a difficult screening case, but the reasoning and disagreement pattern may not always become shared learning. Important judgement moments can remain inside individual memory, informal conversation or isolated records.

The Disagreement Review Workshop creates a structured moment for teams to review difficult AI-human disagreement cases without blaming individual radiologists. It helps move the service from one-off case handling toward shared calibration, better handover, clearer disagreement categories and stronger organisational learning.

## Participants

- Human reader / radiologist
- Senior radiologist / arbitrator
- Trainee / junior reader
- Clinical governance or audit lead
- AI implementation lead or vendor liaison, optional

Suggested size: 4-8 people, 30-45 minutes, monthly or bi-weekly.

## Inputs

1. Non-blame Decision Receipt
2. Learning Memory Card
3. Selected disagreement cases from arbitration

## Workshop flow

1. Case snapshot

A selected anonymised case is introduced with human judgement, AI output, AI confidence and disagreement type.

2. Silent judgement

Participants first make an individual judgement using tokens or buttons: Recall / No recall / Need more evidence. This avoids hierarchy bias before discussion.

3. Reveal arbitration rationale

The final arbitration outcome and reasoning are revealed, including evidence used.

4. Pattern discussion

The team discusses what made the case difficult and assigns a learning tag.

5. Service improvement action

The team decides whether any protocol, handover template, taxonomy, training material or governance question should be updated.

## Artefacts used in the workshop

- Case Snapshot
- Silent Judgement Tokens
- Evidence Strip
- Non-blame Decision Receipt
- Pattern Tag Board
- Action Capture Sheet

## Outputs

- Updated learning tags
- Case added to learning memory
- Calibration material for trainees
- Updated disagreement taxonomy
- Protocol or handover improvements
- Questions for AI governance or model monitoring

## Link to Quiet Review Station

The Quiet Review Station is a lighter everyday extension of the workshop. Selected cases can become "case of the week" prompts in staff rooms or training corners.

This creates a lower-pressure learning route between formal workshops. It lets staff revisit disagreement patterns in small moments, without requiring every learning activity to become a scheduled meeting.

## Prototype value

This is a service design prototype because it tests not only interface content, but roles, rituals, handover, communication, non-blame culture and organisational learning.

It helps the project show that disagreement care is not only a screen problem. The prototype can test how people talk about uncertainty, how records travel between moments, how learning is made shareable, and how accountability can be supported without turning review into blame.
