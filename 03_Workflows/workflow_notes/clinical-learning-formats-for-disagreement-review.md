# Clinical Learning Formats for Disagreement Review

## Purpose

This note collects real-world radiology and breast screening learning formats that could host AI-human disagreement learning without adding a separate break-time activity for radiologists.

Project implication: the [[Disagreement Review Workshop]] should be positioned as an outcome-linked case review format that fits into existing audit, consensus, peer learning, interval cancer review, teaching file, or clinical governance rhythms.

## Key design principle

AI-human disagreement cases should not automatically become learning material at the moment of arbitration. The [[Non-blame Decision Receipt]] is generated immediately after consensus or arbitration, but the [[Calibration Case Card]] should be created only after follow-up outcome is reviewed and the case passes a Learning Value Check.

Suggested service chain:

AI-human disagreement detected  
-> consensus / arbitration review  
-> Non-blame Decision Receipt  
-> follow-up outcome update  
-> outcome-linked Learning Value Check  
-> Calibration Case Card  
-> Disagreement Case Library  
-> existing review / audit / teaching meeting

## Strongest existing fit for this project

The strongest fit is an outcome-linked difficult-case review embedded in existing breast screening audit, interval cancer review, reader QA or consensus learning activity.

Why:

- NHS breast screening already expects double reading, arbitration / consensus in some circumstances, clear documentation of final outcome, audit, interval cancer review and reader QA.
- Interval cancer review already uses follow-up outcome and prior-image review as learning material.
- BSIS / NBSS / FRQA structures already treat sensitivity, specificity, recall rate, false positives, discrepant cancers and interval cancers as quality and learning evidence.
- This lets the prototype avoid adding a separate break-time learning tool. The design layer should prepare better records and learning cards for moments that already exist.

## Proposed meeting flow for our project

Working title: Outcome-linked AI-human disagreement calibration review.

This should be a short segment inside an existing consensus, audit, peer learning, teaching or interval cancer learning meeting.

1. Case selection before the meeting

Cases are selected from the Disagreement Case Library, not directly from all arbitration receipts. Each selected case must have follow-up outcome available and must have passed the outcome-linked Learning Value Check.

2. Case snapshot

The facilitator presents an anonymised Calibration Case Card: Case ID, human judgement, AI evidence layer, final recall decision, follow-up outcome, prior image status and disagreement pattern.

3. Independent judgement moment

Participants briefly review the current and prior images before seeing the final rationale. They choose: recall, no recall, or need more evidence. This mirrors blind or independent judgement logic without extending routine reading time.

4. Reveal consensus / arbitration rationale

The meeting reveals the final recall decision, arbitration rationale, evidence used and follow-up outcome.

5. Pattern discussion

The group discusses why the case was difficult: subtle lesion, prior comparison, confidence gap, lesion location, density, AI mark salience, human uncertainty, workflow pressure, or escalation threshold.

6. Learning tag and action

The case receives or updates learning tags. The group decides whether to create a teaching case, update handover wording, refine disagreement taxonomy, raise an AI governance question, or leave as calibration-only material.

7. Record update

The Calibration Case Card and Disagreement Case Library record are updated with meeting used in, learning point, action owner and review status.

## Realistic Calibration Case Card fields

These fields should be compact enough for a teaching / calibration moment, not a duplicate clinical record.

- Case ID
- Source receipt ID
- Follow-up outcome
- Outcome source: recall assessment / ultrasound / biopsy / pathology / interval cancer review / other
- Human judgement
- AI evidence layer
- AI risk signal or confidence band
- Final recall decision
- Consensus / arbitration rationale
- Disagreement pattern
- Why selected
- Learning focus
- Prior image relevance
- Image set available: current mammogram, prior image, AI mark, human annotation, diagnostic image if appropriate
- Learning tags
- Discussion prompt
- Suggested use: consensus learning, peer learning, interval cancer learning review, trainee calibration, audit, AI governance
- Non-blame note
- Review status
- Date added
- Meeting used in

## How this changes the prototype

The prototype should show a low-burden service layer, not a new review ritual added on top of clinical work.

- Touchpoint 1: soft cue inside the existing review workspace.
- Touchpoint 2: handover and consensus / arbitration support inside the existing review workspace.
- Touchpoint 3: immediate Non-blame Decision Receipt after arbitration.
- Learning layer: outcome-linked Learning Value Check happens later, after follow-up outcome.
- Library layer: only selected Calibration Case Cards enter the Disagreement Case Library.
- Workshop layer: the Disagreement Review Workshop should be framed as a short, reusable segment inside existing peer learning, interval cancer review, audit or teaching formats.
